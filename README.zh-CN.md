# 家康

[English README](./README.md) | 点击左侧链接进入英文版 README

**家康**（English name: **FamilyFit**）是一个面向家庭场景的运动健康小程序高保真 Web Demo，聚焦子女帮助父母制定运动计划、记录运动过程、查看家庭运动情况并形成更好的家庭运动氛围。

## 在线 Demo

[https://family-fitness-miniapp.netlify.app](https://family-fitness-miniapp.netlify.app)

## 项目背景

很多父母有运动需求，但缺少计划、记录和持续反馈；异地子女也很难只靠微信真正帮助父母坚持运动。

本项目尝试从家庭关系出发，验证一个轻量的家庭运动健康闭环：父母可以按计划运动并记录结果，子女和家人可以看到运动情况、进行鼓励，并逐步形成家庭运动氛围。

## 为什么做这个项目

- **微信提醒不够有效**：子女想督促父母运动，但聊天提醒容易被忽略，也很难知道父母有没有真正行动。

- **父母缺少运动记录工具**：父母不是完全不想动，而是缺少简单方式记录做了什么、做了多久、做了多少。

- **运动缺少计划和针对性**：父母需要更适合自身情况、温和、可执行的运动安排，但自己往往没有制定计划和选择动作的能力。

- **现有产品不够匹配**：健身 App 偏年轻人和个人训练，健康 App 偏指标查看，较少覆盖“子女远程帮助父母坚持运动”的场景。

- **家庭参与能提高持续性**：运动被记录、被家人看见、被及时鼓励后，更容易从个人任务变成家庭共同参与的习惯。

## 产品设计思路

项目以 4 个底部 Tab 组织信息架构：

- **运动**：产品第一入口，支持按计划运动和自由运动。
- **家庭**：查看家庭成员状态、家庭动态、家庭运动榜、家庭日历和记录详情。
- **广场**：展示运动、营养、父母健康、动作教学等内容卡片。
- **我的**：维护个人资料、健康档案、计划管理和家庭成员管理入口。

核心流程是：

```text
进入运动首页
-> 选择计划运动或自由运动
-> 添加或查看运动项目
-> 开始运动并记录参数
-> 完成运动并选择发布
-> 家庭动态和家庭日历可见
-> 家庭成员互动和复盘
```

当前版本主要使用前端模拟数据和本地状态来验证交互闭环。健康档案、家庭成员、计划库和内容广场都以原型方式呈现，用于说明正式产品的信息架构和核心体验。

## 核心功能

- 运动首页：进入计划运动、自由运动、最近运动记录和运动项目库。
- 计划模板库：选择家庭常用的有氧、力量、拉伸和恢复训练组合。
- 计划运动：查看今日计划任务，确认动作后开始运动。
- 自由运动：从运动项目库添加动作，配置时长、距离、重量、次数、组数等参数。
- 运动中记录：记录每组完成状态，调整动作参数，完成后生成运动记录。
- 家庭首页：查看成员状态、家庭运动榜、家庭日历预览和运动动态。
- 家庭日历：按日期查看当天完成运动的家庭成员，并进入对应运动记录详情。
- 家庭动态：发布运动记录，查看家庭成员的运动分享、评论和鼓励。
- 健康档案：维护基础资料，记录体重、血压、血糖和心率等健康数据。
- 家庭成员管理：查看家庭成员、处理加入申请和成员健康档案。
- 广场内容：提供运动训练、营养学、日常健康和疾病调控相关内容卡片。

## 当前实现

当前版本是一个可交互的高保真前端原型：

- 使用微信小程序视觉比例设计，可在浏览器中体验。
- 已实现运动、家庭、广场、我的 4 个主 Tab。
- 已实现计划运动、自由运动、运动中记录、完成发布和家庭动态展示等核心流程。
- 已实现家庭日历、记录详情、健康档案、计划管理、成员管理等关键页面入口。
- 数据主要来自前端模拟数据，部分交互使用前端状态。

当前版本暂未接入后端、数据库、支付、真实 AI API、真实微信登录或多端同步。

## 当前技术栈

- **HTML**：页面结构。
- **CSS**：视觉样式、移动端比例、页面布局和交互动效。
- **JavaScript**：页面切换、状态更新和原型交互。
- **Netlify 静态部署配置**：通过 `netlify.toml` 支持静态站点部署。

## 本地运行

项目是静态单页原型，可以直接通过本地静态服务运行：

```bash
python3 -m http.server 5177
```

然后访问：

```text
http://127.0.0.1:5177/index.html
```

也可以直接用浏览器打开 `index.html`，但推荐使用本地服务方式体验。

## 演示流程

1. 打开首页，进入“运动”Tab。
2. 选择“按计划运动”或“自由运动”。
3. 查看计划模板，或从运动项目库添加运动项目。
4. 开始运动，记录动作、组数、重量、次数、时长或距离。
5. 完成运动并选择发布到家庭。
6. 在“家庭”Tab 查看家庭动态、家庭日历和运动记录详情。
7. 在“我的”Tab 查看健康档案和计划管理入口。

## MVP 取舍

- **先验证核心体验**：优先跑通“开始运动 -> 记录 -> 发布 -> 家庭可见”的主路径。
- **先用静态 Web Demo 降低展示成本**：当前版本便于在线访问、快速演示和持续迭代。
- **先保留父母友好体验**：交互和文案尽量降低理解成本，同时保留一定的精细记录能力。
- **先控制风险边界**：当前版本不提供医疗建议，不做疾病诊断或治疗承诺。
- **暂不实现重后端能力**：真实账号、家庭邀请、权限、数据库和云端同步放入后续阶段。

## 后续 Roadmap

- 接入真实用户登录和家庭邀请流程。
- 增加数据库和云端同步，保存运动记录、家庭动态和健康档案。
- 补充隐私授权和家庭成员可见范围设置。
- 接入 AI 计划生成能力，但输出必须是可编辑的建议计划，并包含健康风险提示。
- 迁移到微信小程序原生工程，或使用 Taro / uni-app 等跨端框架。
- 增加家庭周报、分享卡片和演示录屏素材。
- 增加更完整的动作库、内容管理和计划复用能力。
- 根据正式上线要求补充健康数据合规说明。
